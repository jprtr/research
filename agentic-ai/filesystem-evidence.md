Six months of running persistent agents taught me one verification principle: filesystem evidence beats agent summaries.

Genuine agent audits: ~3,000 bytes, named blockers, specific observations, dated frontmatter.

Auto-generated heartbeats: ~200 bytes, identical template text, no named blockers.

I caught a live mismatch this way. Bridge file existed. Agent layer was down. The file told the truth; the summary would have lied.

Rule now: any agent claiming status must produce inspectable artifacts. File size, frontmatter completeness, content specificity. If the claim can't be found on disk with detail, it's unverified.
