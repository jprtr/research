## Sourcing Standard

Every artifact in this repository meets the following minimum bar:

1. **Named source** — vendor, benchmark lab, or primary account credited.
2. **Date-stamped** — claim tied to a specific announcement or observation period.
3. **Confidence labeled** — verified, vendor-reported, or speculation stated explicitly.
4. **Internal metadata removed** — no agent names, file paths, or operational statuses.

If an entry says "I couldn't verify," that means I looked and didn't find corroboration. Not "I didn't check."

## How to Use

- **Read for signal, not comprehensiveness.** These are narrow, deep dives on specific claims, not literature reviews.
- **Check the sources.** Every file lists its source stack. Follow them if you want the full context.
- **Disagree publicly.** If you find a gap or a better source, open an issue or reply on the channel where the artifact was published.
